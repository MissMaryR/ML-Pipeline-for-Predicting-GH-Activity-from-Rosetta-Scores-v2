# Rosetta Docking Data Cleaning Pipeline

A Python pipeline for collecting, cleaning, normalizing, and labeling Rosetta docking score data for GH–oligosaccharide binding prediction.

---

## Overview

This script processes Rosetta docking score files across multiple GH–oligo experimental conditions, cleans and normalizes the data, and outputs a labeled dataset ready for machine learning.

The final output is `8_Book1_final.csv` — one row per GH–oligo pair with normalized docking features and an `active` label (1 = active, 0 = inactive).

## Expected Folder Structure

The script expects the following directory layout where it is run:

```
working_directory/
├── 1_clean_data.py
├── GH1/                ← GH column
│   ├── oligo1/               ← oligo column
│   │   └── Top_PDBs/
│   │       └── glycan_top_fullscores.csv
│   └── oligo2/
│       └── Top_PDBs/
│           └── glycan_top_fullscores.csv
├── GH2/
│   └── ...
```

The `Top_PDBs/glycan_top_fullscores.csv` files are generated by the Rosetta docking scoring script which filters and selects top docking poses.

---

## Usage

Navigate to the directory containing your data folders and run:

```bash
python3 1_clean_data.py
```

## Output Files

Files are numbered in the order they are created so you can follow the pipeline at a glance:

| File | Step | Description |
|------|------|-------------|
| `1_Book1_raw.xlsx` | 0 | Raw combined data from all CSVs |
| `2_Book1_swapped.xlsx` | 1 | SR-corrected data |
| `3_Book1_dropzero.xlsx` | 2 | Fully-zero columns removed |
| `4_Book1_cleaned.xlsx` | 3 | Correlated/redundant columns removed |
| `5_Book1_normalized.xlsx` | 5 | Per-residue and per-subregion normalized features |
| `6_Book1_normalized_cleaned.xlsx` | 7 | Post-normalization correlated columns removed |

| `8_Book1_final.csv` | 9 | Final dataset with active/inactive labels |

`8_Book1_final.csv` is the file to use for model training.

## Pipeline Steps

| Step | Description | Input | Output |
|------|-------------|-------|--------|
| 0 | Collect CSVs into one file | Folder structure | `1_Book1_raw.xlsx` |
| 1 | Swap SR_1/SR_2 if needed | `1_Book1_raw.xlsx` | `2_Book1_swapped.xlsx` |
| 2 | Drop fully-zero columns, report correlations | `2_Book1_swapped.xlsx` | `3_Book1_dropzero.xlsx` |
| 3 | Drop correlated/redundant columns | `3_Book1_dropzero.xlsx` | `4_Book1_cleaned.xlsx` |
| 4 | Redundancy recheck | `4_Book1_cleaned.xlsx` | *(printed report only)* |
| 5 | Normalize scores per residue and per subregion | `4_Book1_cleaned.xlsx` | `5_Book1_normalized.xlsx` |
| 6 | Redundancy recheck post-normalization | `5_Book1_normalized.xlsx` | *(printed report only)* |
| 7 | Drop post-normalization correlated columns, recheck | `5_Book1_normalized.xlsx` | `6_Book1_normalized_cleaned.xlsx` |
| 8 | Aggregate by GH–oligo pair (mean) | `6_Book1_normalized_cleaned.xlsx` | `7_Book1_aggregated.csv` |
| 9 | Add active/inactive labels | `7_Book1_aggregated.csv` | `8_Book1_final.csv` |

---

## Step Details

### Step 0 — Collect
Walks through all subdirectories two levels deep, finds every `Top_PDBs/glycan_top_fullscores.csv`, and combines them into a single Excel file. Adds `GH` and `oligo` columns based on the folder names.

### Step 1 — Swap SR_1/SR_2
Ensures the smaller subregion is always labeled SR_1 and the larger SR_2. Swaps all associated `SR_1_*` and `SR_2_*` columns row-wise where needed.

### Step 2 — Drop Zero Columns
Removes any numeric column that is 100% zeros across all rows. Also prints all column pairs with >= 95% Pearson correlation for review before the next step.

### Step 3 — Drop Correlated Columns
Drops manually curated redundant columns identified from the step 2 correlation report. These include duplicate score terms, raw Rosetta energy terms already captured in `total_score`, and uninformative features.

### Step 4 — Recheck
Prints any remaining highly correlated pairs and zero columns after step 3 to confirm the data is clean before normalization. Does not modify any files.

### Step 5 — Normalize
Normalizes global score columns by residue count (per GH) and SR-specific columns by their subregion size. Raw columns are dropped immediately after normalization to avoid keeping redundant pairs. Residue counts are hardcoded per GH and should be updated to match your own sequences:

```python
residue_counts = {
    'GH1': 1000,
    'GH2': 800,
    # add your GH names and residue counts here
}
```

### Step 6 — Recheck Post-Normalization
Same correlation/zero check as step 4, run on the normalized data. Does not modify any files.

### Step 7 — Drop Post-Normalization Redundancies
Drops any remaining correlated columns that only appear after normalization. Runs a final recheck and saves a clean normalized file.

### Step 8 — Aggregate
Groups all rows by GH–oligo pair and takes the mean of all numeric columns, reducing multiple docking poses per pair down to a single representative row.

### Step 9 — Label Active/Inactive
Adds an `active` column (1 = active, 0 = inactive) based on known experimental activity. Update the `active_pairs` dictionary to match your own data:

```python
active_pairs = {
    'oligo1': {'GH1', 'GH2'},
    'oligo2': {'GH1'},
    # add your oligo names and active GH sets here
}
```

All GH–oligo pairs not listed are labeled inactive.

## Customizing for Your Data

Two places in the script need to be updated when running on a new dataset:

**1. Residue counts** — in `step5_normalize()`:
```python
residue_counts = {
    'GH1': 1000,   # replace with your GH names and residue counts
    'GH2': 800,
}
```

**2. Active pairs** — in `step9_label_active()`:
```python
active_pairs = {
    'oligo1': {'GH1', 'GH2'},   # replace with your oligo names and active GH sets
    'oligo2': {'GH1'},
}
```

---

## Notes

- Intermediate files are preserved at each step so you can inspect the data or re-run from any point in the pipeline.
- Steps 4 and 6 are report-only steps — they print correlation information but do not save any files.
- To add new columns to drop, edit the `columns_to_drop` list in `step3_drop_correlated()` or `step7_drop_and_recheck()`.
