# Rosetta Docking Data Cleaning Pipeline

A Python pipeline for collecting, cleaning, normalizing, and labeling Rosetta docking score data for GH3–oligosaccharide binding prediction.

---

## Overview

This script processes Rosetta docking score files (`.sc`) across multiple GH3–oligo experimental conditions, cleans and normalizes the data, and outputs a labeled dataset ready for machine learning.

The final output is `Book1_aggregated.csv` — one row per GH3–oligo pair with normalized docking features and an `active` label (1 = active, 0 = inactive).

## Expected Folder Structure

The script expects the following directory layout where it is run:

```
working_directory/
├── 1_clean_data.py
├── oligo_name_1/          ← oligo column
│   ├── GH3_name_1/        ← GH3 column
│   │   └── Top_PDBs/
│   │       └── glycan_top_fullscores.csv
│   └── GH3_name_2/
│       └── Top_PDBs/
│           └── glycan_top_fullscores.csv
├── oligo_name_2/
│   └── ...
```

The `Top_PDBs/glycan_top_fullscores.csv` files are generated by the [Rosetta docking scoring script](score_top_pdbs.py) which filters and selects top docking poses.

---

## Usage

Navigate to the directory containing your data folders and run:

```bash
python3 1_clean_data.py
```

## Pipeline Steps

| Step | Description | Input | Output |
|------|-------------|-------|--------|
| 0 | Collect CSVs | Folder structure | `Book1.xlsx` |
| 1 | Swap SR_1/SR_2 if needed | `Book1.xlsx` | `Book1_swapped.xlsx` |
| 2 | Drop fully-zero columns, report correlations | `Book1_swapped.xlsx` | `Book1_swapped.xlsx` |
| 3 | Drop correlated/redundant columns | `Book1_swapped.xlsx` | `Book1_cleaned.xlsx` |
| 4 | Redundancy recheck | `Book1_cleaned.xlsx` | *(printed report)* |
| 5 | Normalize scores per residue and per subregion | `Book1_cleaned.xlsx` | `Book1_normalized.xlsx` |
| 6 | Redundancy recheck post-normalization | `Book1_normalized.xlsx` | *(printed report)* |
| 7 | Drop post-normalization correlated columns, recheck | `Book1_normalized.xlsx` | `Book1_normalized.xlsx` |
| 8 | Aggregate by GH3–oligo pair (mean) | `Book1_normalized.xlsx` | `Book1_aggregated.csv` |
| 9 | Add active/inactive labels | `Book1_aggregated.csv` | `Book1_aggregated.csv` |

---

## Step Details

### Step 0 — Collect
Walks through all subdirectories two levels deep, finds every `Top_PDBs/glycan_top_fullscores.csv`, and combines them into a single Excel file. Adds `GH3` and `oligo` columns based on folder names.

### Step 1 — Swap SR_1/SR_2
Ensures the smaller subregion is always labeled SR_1 and the larger SR_2. Swaps all associated `SR_1_*` and `SR_2_*` columns row-wise where needed.

### Step 2 — Drop Zero Columns
Removes any numeric column that is 100% zeros across all rows. Also prints all column pairs with >= 95% Pearson correlation for review.

### Step 3 — Drop Correlated Columns
Drops manually curated redundant columns identified from the correlation report. These include duplicate score terms, raw Rosetta energy terms already captured in `total_score`, and uninformative features.

### Step 4 — Recheck
Prints any remaining highly correlated pairs and zero columns after step 3 to confirm the data is clean before normalization.

### Step 5 — Normalize
Normalizes global score columns by residue count (per GH3) and SR-specific columns by their subregion size. Raw columns are dropped after normalization to avoid keeping redundant pairs. Residue counts are hardcoded per GH3:

| GH3 | Residue Count |
|-----|--------------|
| 39  | 1001 |
| 41  | 801  |
| 47  | 994  |
| 76  | 857  |
| B3  | 740  |
| B6  | 817  |
| B9  | 945  |
| I1  | 829  |
| I2  | 823  |
| W0  | 1080 |

### Step 6 — Recheck Post-Normalization
Same correlation/zero check as step 4, but run on the normalized data.

### Step 7 — Drop Post-Normalization Redundancies
Drops any remaining correlated columns that only appear after normalization (e.g. `fa_dun_per_res`, `tot_total_neg_charges_per_res`). Runs a final recheck.

### Step 8 — Aggregate
Groups all rows by GH3–oligo pair and takes the mean of all numeric columns, reducing multiple docking poses per pair down to a single representative row.

### Step 9 — Label Active/Inactive
Adds an `active` column (1 = active, 0 = inactive) based on known experimental activity:

| Oligo | Active GH3s |
|-------|-------------|
| CL3   | B9, B3, 76, 41 |
| CR3   | B9, B3, I2, 41, 76, 47 |
| XY3   | B9 |
| H2B   | B9 |
| H3B   | B9, B3 |

All other GH3–oligo pairs are labeled inactive.

## Output Files

| File | Description |
|------|-------------|
| `Book1.xlsx` | Raw combined data |
| `Book1_swapped.xlsx` | SR-corrected data |
| `Book1_cleaned.xlsx` | Redundant columns removed |
| `Book1_normalized.xlsx` | Per-residue and per-subregion normalized features |
| `Book1_aggregated.csv` | Final dataset — one row per GH3–oligo pair with active labels |

`Book1_aggregated.csv` is the file to use for model training.

---

## Notes

- The intermediate `.xlsx` files are preserved at each step so you can inspect the data or re-run from any point in the pipeline.
- If a `Top_PDBs` folder already exists for a condition, it will be included automatically — re-run the collection script to update.
- To add new GH3 residue counts or update active/inactive labels, edit the `residue_counts` dictionary in `step5_normalize()` and the `active_pairs` dictionary in `step9_label_active()`.
